# Goals Feature — Design Discussion

## Open Questions

### Scope
- Goals per account ("save $10K in Emergency fund")?
- One overall goal ("save $50K total")?
- Both — let users set either?

### Target Date
- Required or optional?
- If optional, some people just want a number to hit without a deadline
- If there's a date, show projected completion? ("At your current rate, you'll hit this by October")

### Visual / Placement
- Progress ring? Progress bar? Percentage overlay on existing chart?
- Where does it live — main screen always visible, or behind a tap?
- Subtle or bold? (Should it dominate the screen or be a supporting element?)

### What happens when you hit the goal?
- Confetti? (Already have the confetti system from easter egg)
- Celebration screen?
- Archive it and set a new one?
- Badge/trophy that persists?

### Data Model
- Goal amount (required)
- Target date (optional)
- Linked to: one category OR "total" across all accounts
- Do goals persist after being hit, or disappear?
- Multiple goals at once, or one at a time?

### Pro Gate
- Where does the paywall go?
- Free users see the Goals button but get a Pro unlock prompt?
- Or free users get 1 goal, Pro gets unlimited?
